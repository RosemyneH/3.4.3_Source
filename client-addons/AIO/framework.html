AIO — All-In-One Bridge

Bi-directional channel between the Trinity 3.4.3 worldserver and the WoW Classic client addon, using whisper addon messages.

Wire format

Prefix: AIO
Body:   handlerName + TAB + payload
Payload: key=value;key2=value2

Layers

LayerLocationRole
Server coresrc/server/game/AIO/Handler registry, codec, send
Server scriptssrc/server/scripts/Custom/custom_aio.cppBuilt-in handlers, login flow
Client coreclient-addons/AIO/Core/Bus, codec, events, modules
Client handlersclient-addons/AIO/Handlers/core.* and dev.* protocols
Client modulesclient-addons/AIO/Modules/Your gameplay features

Core handlers

HandlerDirectionPurpose
core.handshakeC→SClient version on login
core.readyS→CRealm, build, capabilities
core.pingC→SLatency / link check
core.pongS→CServer uptime reply
core.notifyS→CColored chat line
dev.localhostS→CLocal dev panel data

Client API

AIO.Register("my.handler", function(raw, data) end)
AIO.Send("my.handler", "payload")
AIO.SendTable("my.handler", { key = "value" })
AIO.On("READY", function(serverData) end)
AIO.Module.Define("name", function(mod) end)

Server API

sAIO->RegisterHandler("my.handler", [](Player* p, std::string_view payload) { });
sAIO->SendTable(player, "my.handler", { {"key", "value"} });

Deploy

./scripts/prepare-client.sh   # installs addon + Config.wtf
